# 双人对决 · 云端接力（跳一跳）PRD

> 第二款 3D 双人小游戏，与「云上之巅」并列，挂在大厅"GAME MODE"卡片下。

## 1. 产品概述

### 1.1 一句话定位
**回合制 · 共享赛道 · 骰子驱动的双人 3D 跳一跳竞速游戏。**

### 1.2 核心体验
两位玩家共享一条由浮空板块连接的"云端阶梯"，按回合制轮流投掷骰子。骰子结果决定本回合的事件——前进若干格、获得 buff、触发陷阱、或对对手施加 debuff。无需操作角色，所有动作由系统驱动，玩家只需投骰、看结果、互相干扰。**先抵达终点板的玩家获胜。**

### 1.3 与「云上之巅」的差异

| 维度 | 云上之巅（已实现） | 云端接力（本 PRD） |
|------|-------------------|---------------------|
| 玩法节奏 | 实时操作 / 双人同屏 | 回合制 / 轮流出手 |
| 控制方式 | 一台键盘双控（WASD / 方向键） | 单键投骰（Space / Enter）|
| 物理 | rapier 真实物理 + 碰撞 | 抛物线动画（无物理） |
| 节奏 | 紧张刺激 / 高手速 | 策略博弈 / 看运气和算计 |
| 适合人群 | 反应党 / 操作党 | 全年龄 / 派对 / 不擅手柄 |

## 2. 玩法详述

### 2.1 赛道
- **总长**：20 块板（编号 0 ~ 19，0 为起点，19 为终点旗板）
- **板间距**：固定 2.5m（视觉等距）
- **赛道形态**：略蜿蜒（每 5 块板换一次方向，避免一条直线无聊）
- **板块外观**：复用浮空岛风格（顶面草地绿，底部岩石紫），终点板镶金边 + 旗杆

### 2.2 回合流程

```
[当前回合方] ─ 按 Space/Enter 投骰 ─→ [骰子动画 ~1.5s] ─→ [事件触发动画 ~1.5s] ─→ [切换回合]
                                          ↓
                                   显示事件结果横幅
```

每一回合 ≈ 3-4 秒，节奏轻快不拖沓。

### 2.3 骰子事件池（6 面骰）

骰子有 6 种结果，每次随机出现一种：

| 面 | 图标 | 名称 | 效果 |
|----|------|------|------|
| 1  | 🏃 | 小步前进 | 前进 **1 格** |
| 2  | 🏃‍♂️ | 大步前进 | 前进 **3 格** |
| 3  | ⚡ | 闪电加速 | 前进 **2 格**，下回合再 **+1** 格 |
| 4  | 🛡️ | 圣盾护持 | 原地不动，获得 **护盾**（抵御 1 次对手 debuff）|
| 5  | 💥 | 推搡攻击 | 自己前进 **1 格**，对手 **后退 2 格**（有护盾则消耗护盾不后退）|
| 6  | 🔄 | 命运互换 | **双方位置互换**（仅当对手领先时触发；落后时改为前进 2 格）|

> 设计原则：6 种事件全部"看得见、听得懂"，无需教程。负面事件（推搡 / 互换）受护盾保护，避免完全靠运气。

### 2.4 胜负判定
- 哪位玩家先**抵达或越过编号 19 的终点板** → 立刻获胜
- 越过终点的多余格数不浪费，直接结算
- 没有平局（共享赛道、回合先后明确）

### 2.5 掉落处理
- 跳一跳玩法本身不会"自然掉落"（每次跳跃由系统动画驱动，必定落在目标板上）
- 仅当遇到"推搡后退"导致编号 < 0 时，回到 0 号板（不淘汰）

### 2.6 操作按键

| 玩家 | 投骰键 |
|------|--------|
| P1 | **Space** |
| P2 | **Enter** |

非回合方按键无效；回合方按键触发骰子。**支持鼠标点击屏幕上的"ROLL"按钮**作为备选。

## 3. UI / UX

### 3.1 场景视觉
- **背景**：复用 `SkyScene`（蓝色渐变天空 + 飘云 + 浮空感）
- **赛道**：20 块板沿 Z 方向延伸（蜿蜒），每块板下方有岩石支柱
- **角色**：复用现有 `Player` 胶囊体外观（P1 青蓝 / P2 粉色），但**无物理**，纯动画驱动

### 3.2 HUD 元素

```
┌─────────────────────────────────────────────────┐
│  [P1: 5/20 ████░░░░]   VS   [P2: 3/20 ██░░░░]  │  ← 顶部双方进度
│                                                  │
│            [当前回合：P1 的回合]                  │  ← 顶部中间提示
│                                                  │
│                                                  │
│                  ╭─────────╮                     │
│                  │   🎲    │  ← 中央 3D 骰子      │
│                  ╰─────────╯                     │
│                                                  │
│       [按 SPACE 投掷骰子 / 点击屏幕]              │  ← 引导
│                                                  │
│   [P1 buff: 🛡️]                  [P2 buff: ⚡]   │  ← 底部 buff 槽
└─────────────────────────────────────────────────┘
```

### 3.3 事件提示横幅
骰子停止后，屏幕中央弹出**横幅**（1.5 秒）显示事件名称与图标：

```
       💥 推搡攻击
   P1 +1  ·  P2 -2
```

### 3.4 颜色 & 字体
- 完全复用现有 `theme.css` 设计 token
- 主标题字体 Orbitron（已加载）
- P1 = `--p1-color`（青蓝）、P2 = `--p2-color`（粉色）
- 当前回合方的进度条/卡片**发光高亮**

## 4. 技术方案

### 4.1 目录结构

```
frontend/src/
├── engine_jump/                # 跳一跳引擎核心
│   ├── types.ts                # 类型：玩家状态、骰子事件、回合
│   ├── constants.ts            # 常量：板数、间距、骰子事件配置
│   └── jumpStore.ts            # zustand 全局状态
├── components_jump/            # 跳一跳 3D 组件
│   ├── JumpTrack.tsx           # 赛道（20 块板 + 终点旗）
│   ├── JumpPlayer.tsx          # 角色（抛物线跳跃动画）
│   ├── Dice.tsx                # 3D 骰子（摇骰动画 + 数字面）
│   ├── JumpFollowCamera.tsx    # 跟随相机（聚焦当前回合方）
│   ├── JumpCanvas.tsx          # R3F Canvas 根
│   └── JumpHUD.tsx + .css      # HUD：进度条、回合提示、骰子按钮、buff
└── pages/
    └── JumpGamePage.tsx        # 页面 + 路由 /jump/local
```

### 4.2 关键技术点

1. **无物理**：跳跃动画用 `useFrame + lerp + 抛物线 y = -4*(t)*(t-1)*h`（t∈[0,1]），稳定且与板对齐
2. **3D 骰子**：6 个面贴不同 emoji 数字纹理（或使用 `<Text>` 直接渲染），摇动用四元数 lerp
3. **回合制状态机**：`phase: idle | rolling | resolving | finished`
4. **回合切换**：事件解决后 800ms 自动切换到对方
5. **复用现有资产**：
   - `SkyScene`（直接 import）
   - `theme.css` 颜色变量
   - `engine3d/constants.ts` 的 P1_COLOR / P2_COLOR
   - `pages/ResultPage`（payload 结构兼容）

### 4.3 状态机示意

```
        ┌──────┐  双方 READY    ┌──────────┐
        │ idle │ ─────────────→ │ p1_turn  │
        └──────┘                └────┬─────┘
                                     │ Space
                                     ▼
                              ┌─────────────┐
                              │   rolling   │  骰子动画 1.5s
                              └──────┬──────┘
                                     │ 骰子停止
                                     ▼
                              ┌─────────────┐
                              │  resolving  │  事件横幅 + 跳跃动画
                              └──────┬──────┘
                                     │ 800ms 后
                            ┌────────┴────────┐
                            │                 │
                            ▼                 ▼
                      ┌──────────┐      ┌──────────┐
                      │ p2_turn  │      │ finished │
                      └────┬─────┘      └──────────┘
                           │ Enter（同上）
```

### 4.4 与现有系统的集成

| 集成点 | 做法 |
|--------|------|
| 路由 | 在 `App.tsx` 增加 `/jump/:roomId` → `JumpGamePage` |
| 大厅 | `LobbyPage` 的 `mode-grid` 增加一张 ModeCard："云端接力（跳一跳）" |
| 结算 | 复用 `ResultPage`，写入 `sessionStorage.lastGameResult`，跳转 `/result/local` |
| 主页 | 暂不改（home 副标题保持"云上之巅"为主推） |

## 5. MVP 范围 & 时间预估

### MVP 必须有
- ✅ 20 板赛道（直线 / 微蜿蜒）
- ✅ 完整 6 种骰子事件
- ✅ 双人回合制 + 投骰动画
- ✅ 跳跃抛物线动画
- ✅ HUD 进度条 + 回合提示 + 事件横幅
- ✅ 大厅入口 + 结算页接入

### MVP 暂不做（后续迭代）
- ❌ 骰子贴图（用纯色 + Text 数字）
- ❌ 音效
- ❌ 联机
- ❌ 特殊地形（加速带、随机传送门等地图机制，留到 V2）
- ❌ 角色技能选择（每个角色不同骰子池，留到 V3）

### 工时预估
- 引擎核心：30 分钟
- 3D 组件（赛道 + 角色 + 骰子）：60 分钟
- HUD + 页面：30 分钟
- 联调 + lint：15 分钟
- **合计 ~2 小时**
