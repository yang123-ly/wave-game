| :--- | :--- | :--- | :--- |

---

## 1. 项目概述

### 1.1 产品背景
原版「双人对决·极速任务王」（2D 反应力分屏）已完成 MVP，但用户希望体验更沉浸的 **3D 闯关玩法**。沿用"双人本地同屏对决"基因，将"反应力比拼"升级为"3D 跑酷+互动对抗"，让游戏从"考反应"进化为"考反应 + 走位 + 心机"。

### 1.2 产品定位
- **类型**：3D 双人对抗闯关 / 休闲竞技 / 派对游戏
- **平台**：Web (React + Three.js)，桌面浏览器优先；移动端虚拟摇杆作为 Phase 2
- **核心体验**：一台键盘双人同屏，3D 共享赛道，**推撞 + 抢道具 + 最先抵达终点**
- **目标用户**：朋友/情侣聚会、办公室摸鱼五分钟一局、休闲玩家
- **关键词**：☁️ 浮空岛屿、明亮卡通、治愈奇幻、推搡心机、跳板跑酷

### 1.3 核心价值主张
| 价值 | 描述 |
| :--- | :--- |
| **同屏对抗的快乐** | 一台键盘双人，物理推撞带来面对面笑声 |
| **3D 沉浸感** | R3F 渲染的浮空岛屿、动态云海、卡通材质 |
| **公平且有变数** | 共享赛道 = 起点相同；道具/陷阱 = 局势随时翻转 |
| **极短单局** | 60-120 秒一局，符合碎片化娱乐场景 |

## 2. 核心玩法 (Core Gameplay)

### 2.1 模式：共享赛道对抗闯关
- **场景**：单一 3D 浮空岛屿赛道，两名玩家共享同一镜头空间，从同一起点出发。
- **目标**：最先抵达终点平台的玩家获胜。
- **关键规则**：
  1. 玩家角色之间**可发生物理碰撞**（推开、撞飞、阻挡）。
  2. 赛道上散布 **加速 / 护盾 / 击退波** 三类道具，先到先得。
  3. 掉出赛道 → 触发**复活机制**：2 秒后在最近的检查点（Checkpoint）重生。
  4. 移动/旋转机关（齿轮、摆锤、消失平台）周期性运作，撞到/被砸 → 击退 + 短暂硬直。

### 2.2 操作（一台键盘双人）
| 操作 | 玩家 1 | 玩家 2 |
| :--- | :--- | :--- |
| 前进 | `W` | `↑` |
| 后退 | `S` | `↓` |
| 左转 / 左移 | `A` | `←` |
| 右转 / 右移 | `D` | `→` |
| 跳跃 | `Space` | `Enter` |
| 使用道具 | `F` | `Shift` |

> **设计理由**：左手区与方向键区物理分离，双人同时按键无冲突；跳跃/技能键避开热门组合（Alt/Ctrl）以兼容浏览器。

### 2.3 摄像机方案（MVP）
- **单一固定第三人称侧后跟随镜头**，俯瞰整条赛道，自动 framing 让两名玩家始终可见。
- 当两人距离 > 阈值时，镜头自动拉远（动态 FoV / 距离插值）。
- **不做分屏**（分屏 3D 工作量过大且小屏阅读差）。

### 2.4 赛道结构（MVP 1 关：「云端竞速」）
赛道由 6 段（Segment）串联，总长 ~80 米：

| 段 | 名称 | 元素 | 互动点 |
| :--- | :--- | :--- | :--- |
| 1 | 启航云台 | 平直起点，3 秒倒计时 | — |
| 2 | 跳板桥 | 3 个间隔逐渐拉大的浮空跳板 | 推下对手得分 |
| 3 | 齿轮回廊 | 2 个旋转齿轮+夹缝通过 | 撞到 → 击退 |
| 4 | 道具岛 | 中央密集 3 个道具点 | 抢道具 |
| 5 | 消失平台阵 | 6 块周期消失的方砖 | 节奏感闯关 |
| 6 | 终点云顶 | 大平台 + 终点旗 | 触碰旗子即胜利 |

### 2.5 道具系统
| 图标 | 道具 | 效果 | 持续 |
| :--- | :--- | :--- | :--- |
| ⚡ | 加速冲刺 | 移动速度 +60% | 3 秒 |
| 🛡️ | 护盾 | 免疫一次推撞 / 击退 | 至消耗 |
| 💥 | 击退波 | 主动技能：向前方 4m 范围释放击退 | 即时 |

道具拾取即生效（加速/护盾），击退波进入"待释放"槽，按 `F` / `Shift` 触发。

### 2.6 胜负与复活
- **胜利**：率先触碰终点旗。
- **复活**：掉出赛道下方 Y < -10 → 镜头淡黑 → 2 秒后回到上一个检查点（每段起点为检查点）。
- **超时**：对局上限 180 秒，仍未分出胜负 → 触发"圣杯之雨"（赛道随机抹除平台），逼迫快速决出。

## 3. 视觉与美术 (Art Direction)

### 3.1 色彩
延续现有"暗黑霓虹电竞风"作为大厅/UI 基调，**对局内场景切换为浮空岛屿亮色**：
- 天空：渐变青蓝 `#87ceeb` → `#a4c8f0`（顶部）→ `#fff7c8`（地平线）
- 云：纯白 `#ffffff` 半透明
- 浮岛草地：薄荷绿 `#7ee787`
- 浮岛侧面/岩石：薰衣草紫 `#c4b5fd`
- P1 角色：青色 `#00d4ff`（与大厅 P1 色一致）
- P2 角色：品红 `#ff4d8d`（与大厅 P2 色一致）

### 3.2 模型风格
- **低多边形 (Low-poly)** + 卡通描边，全部用 R3F primitive / drei 几何体合成，无外部模型依赖。
- 角色：胶囊体 + 圆头球，眼睛用 2 个小球点缀；不做骨骼动画，靠简单的 squash/stretch 表达跳跃。
- 跳板：六边形棱柱 + 微微发光描边。
- 道具：旋转浮动的发光多面体（八面体 / 二十面体）。

### 3.3 后处理
- **Bloom**（适度发光）用于道具与角色描边
- **天空盒**：drei `<Sky>` 组件
- **环境光**：HemisphereLight（天蓝 + 草绿）

## 4. 技术架构

### 4.1 前端栈
| 类别 | 选型 |
| :--- | :--- |
| 框架 | React 18 + TypeScript（沿用） |
| 3D 渲染 | **Three.js + @react-three/fiber + @react-three/drei** |
| 物理 | **@react-three/rapier** (WASM, 高性能, R3F 官方推荐) |
| 状态管理 | **zustand**（轻量、R3F 兼容性好，对局状态独立于 React 渲染） |
| 路由/构建 | 复用现有 CRA + react-router |

### 4.2 模块划分
```
frontend/src/
├── engine3d/
│   ├── gameStore.ts          # zustand 全局对局状态
│   ├── useKeyboardControls.ts# 双玩家键盘输入 hook
│   ├── constants.ts          # 物理参数、跑酷常量
│   └── types.ts              # 类型定义
├── components3d/
│   ├── GameCanvas.tsx        # R3F <Canvas> 根
│   ├── SkyScene.tsx          # 天空盒、光照、云
│   ├── Player.tsx            # 玩家角色 + 物理 RigidBody
│   ├── Track.tsx             # 整条赛道（6 段 segment 拼接）
│   ├── segments/
│   │   ├── StartSegment.tsx
│   │   ├── JumpBridgeSegment.tsx
│   │   ├── GearSegment.tsx
│   │   ├── PowerUpSegment.tsx
│   │   ├── VanishingSegment.tsx
│   │   └── GoalSegment.tsx
│   ├── PowerUp.tsx           # 道具（加速/护盾/击退）
│   ├── HUD.tsx               # 头顶名字、血条、道具图标、终点距离
│   └── FollowCamera.tsx      # 双玩家自适应相机
└── pages/
    └── GamePage.tsx          # 接入 GameCanvas + 入场/结算流转
```

### 4.3 物理与碰撞
- 玩家：`<CapsuleCollider>` + `RigidBody (dynamic, lockRotations)`
- 赛道：`RigidBody (fixed)` + 形状碰撞器
- 道具：`Sensor`（不阻挡，仅触发拾取事件）
- 击退波：球形 OverlapBox + 计算方向，对其他 Player 的 RigidBody applyImpulse

### 4.4 性能目标
- 60 FPS @ 1080p（中端笔记本核显达标）
- 总三角面数 < 30k
- 物理步长固定 60Hz

## 5. 数据与后端

### 5.1 复用现有后端
- 用户体系、登录注册：**完全沿用**
- 排行榜：复用 `/api/users/leaderboard`，**新增 `gameMode` 区分维度**（默认 sky 模式）

### 5.2 新增字段（Phase 2，本期可不实现）
- `user.best_time_sky` — 云上之巅最快通关时间
- `match_record` — 单局记录（玩家、对手、用时、胜负、关卡）

### 5.3 本期数据流（Phase 1 MVP）
- 对局开始/结束都在前端进行
- 结算后将 `{winner, durationMs, p1Time, p2Time}` 写入 `sessionStorage` 给 ResultPage
- 暂不上报后端（避免 schema 改动）

## 7. 非功能性需求

| 项 | 要求 |
| :--- | :--- |
| 加载时间 | 从大厅点击进入 → 可操作 < 3 秒（含 R3F 资源就绪） |
| 帧率 | ≥ 50 FPS（最低中端笔记本核显） |
| 输入延迟 | 按键到角色反应 < 50ms |
| 兼容性 | Chrome / Edge / Safari 最新 2 个版本 |
| 横竖屏 | 桌面端为主；移动端竖屏给出"请使用桌面"提示 |
| 容错 | WebGL 不可用 → fallback 提示页 |

## 9. 风险与对策

| 风险 | 对策 |
| :--- | :--- |
| R3F + rapier WASM 首次加载慢 | Canvas 内提供 `<Suspense>` + 进度条；预热 chunk |
| 物理推撞参数难调（一击就飞 / 推不动） | 在 `constants.ts` 统一 tweakable 参数，便于联调 |
| 双玩家共享键盘冲突 | 按键完全分离左右手区域；KeyboardEvent.code 而非 key 避免输入法干扰 |
| WebGL 不可用 | `useThree` 失败时回退到"环境不支持"提示页 |
| 单关枯燥 | MVP 阶段限定 60s 一局，重玩压力小；通过道具/推撞制造变化 |
